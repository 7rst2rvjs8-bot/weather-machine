The method

Built like a forecast model.

Weather prediction didn’t get accurate from one brilliant model. It got accurate from plain ones that wrote down where they were wrong and added the single fix that would’ve caught each mistake — repeated for decades. This system works the same way, pointed at markets. New to this idea? Start here.

01  /  The engine

It improves by fixing its own mistakes.

The goal isn’t one clever strategy; it’s a collection of independent strategies, each correcting its own errors over time. Today’s strategy is treated, on purpose, as the crude first version — something to improve, not to defend. Each later addition has to catch a specific, recurring mistake the system keeps making. Getting more capable over time is the goal; the discipline below is what keeps that from turning into self-deception.

A fix earns its place only by catching a real, named mistake on data the system has never seen. Build all the machinery you want — make no decision against old data.

02  /  The loop

The same five steps, turned over and over.

01 · TRADE

Make the move

A real trade, or a recorded one — a signal acted on, or logged as one that would have fired.

02 · LOG THE MISS

Write down the mistake

The mistakes are the fuel. Nothing improves without an honest record of being wrong.

03 · PROPOSE A FIX

One correction

The single change that would have caught that exact miss — a fresh idea, not a tweak.

04 · TEST FORWARD

On unseen days

Try it on days it has never touched. Never rescued by re-reading the past.

05 · KEEP / DROP

Earned, or gone

Cheap to drop, hard to keep. It survives only provisionally; one failure ends it.

// No named mistake → no fix → it doesn’t belong on the roadmap. The loop can’t turn faster than real trading days arrive — the limit is the calendar, not effort.

03  /  The search rule

Look widely. Bet narrowly.

Two activities are kept strictly apart, because mixing them is exactly how a system fools itself: learning from lots of ideas (free, do it widely), and choosing which idea to fund with real money (expensive, do it carefully). Every candidate gets watched and learned from; almost none get funded.

[ Learning — wide, free ]

Watch everything

Every idea is recorded and studied. An idea that’s sound but too costly to trade is a useful finding, not a silent reject.

[ Choosing — narrow, costly ]

Fund almost nothing

Promoting anything to real money is a separate, pre-committed test with a high bar. Watching informs that test without being it.

The wide search feeds learning — never funding directly. Collapse the two and you sneak luck back in at full scale.

The load-bearing line
04  /  Why the discipline is strict

The past is a budget you can only spend down.

Every test run against the same slice of market history uses up one of a limited number of honest tries — and most are already spent. Test it enough and any system will look brilliant on paper and behave like a coin flip live. So the rule is never “keep it simple.” The aim is capable and proven, built forward — and honesty about that is the only real defense.
