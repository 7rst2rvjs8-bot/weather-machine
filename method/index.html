The method

Built like a forecast model.

Weather prediction didn't get accurate from one genius model. It got accurate from crude models that logged where they were wrong and added the one term that would have caught each miss — iterated for decades. This system is the same architecture, applied to markets.

01  /  The engine

An error-correction engine, not a fixed strategy.

The goal isn't one clever model; it's a book of independent strategies, each running its own error-correction loop. The current champion is deliberately framed as the crude first model. Every later addition is a candidate correction term — added not as a feature off a menu, but because it catches a specific, recurring class of miss the system currently whiffs on. Complexity is the goal; the discipline below is what keeps it from becoming a fake version of itself.

A correction term earns its place only by catching a real, named miss in data the system has never seen. Build all the plumbing you want — make no decision against old data.

02  /  The loop · one error-correction cycle

Every strategy is the same five-step loop, turned over and over.

01 · TRADE

Take the position

Live, or shadowed — a signal acted on, or logged as one that would have been.

02 · LOG THE MISS

Record where it was wrong

The miss is the fuel. Rich miss-logging is the highest-value thing the live system produces.

03 · PROPOSE THE TERM

One correction

The single term that would have caught that named miss — a fresh hypothesis, not a tweak.

04 · FORWARD-TEST

On unseen days

Shadow it forward against data it has never touched. Never rescued by re-reading the past.

05 · PROMOTE / KILL

Earned, or retired

Cheap to kill, hard to promote. It survives only provisionally; a failure kills it definitively.

// No named miss → not a correction term → it does not belong on the roadmap. The loop cannot turn faster than real market days accrue: the binding constraint is the calendar, not effort.

03  /  The search doctrine

Wide over the unknown. Narrow only at the evidenced gate.

The hardest discipline is keeping two activities strictly apart — because collapsing them is exactly how a system fools itself.

[ Characterization — wide, free ]

Learning

Every candidate enters as data: its properties logged per signal, never gated on. A signal that's sound but whose costs eat it is a finding, not a silent rejection.

[ Selection — narrow, costly ]

Promotion

Promoting anything to live capital is a separate, pre-registered gate under full multiple-comparisons deflation. The wide net informs this gate without being it.

The wide net feeds learning — never feeds selection directly. Characterization is free; comparison costs power; they live at different layers. Collapse them and the selection bias reappears at universe scale.

The load-bearing line
04  /  Why the discipline is non-negotiable

The past is a budget you can only spend down.

Every test run against the fixed historical window spends one comparison from a finite budget — and roughly twenty are already gone. Mine it hard enough and any system looks omniscient in backtest and behaves like a coin flip live. So validation discipline is never argued as "keep it simple." The target is complex-and-validated, built forward. Simplicity is never the case; honest forward measurement is. One validated term at a time.
